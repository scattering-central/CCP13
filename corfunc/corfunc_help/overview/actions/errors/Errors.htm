4.4 Errors and Warnings

Figure 1: Warning dialog: non-fatal
 

Figure 2: Error dialog: fatal
 

Figure 3: Warning messages detailed in the Message window
 

During any of the processes Extrapolate, Transform or Extract parameters, which can be executed by selecting the required item from the Actions menu, the user may be presented with a warning (see Figure 1) or error dialog (see Figure 2). In both cases, the warning or error message will be recorded at the point it occurred in the Message window (see Figure 3). A warning message indicates a non-fatal error i.e. the process has been completed despite the warning. An error dialog indicates that the process was terminated before completion and must be executed again, once the necessary changes have been made to the input parameters.

The most common warnings and errors occur during Extrapolation and are described below:
 

4.4.1 Common Warnings
 
Warning message Cause Remedy
Ran out of iterations before convergence Maximum number of tail-fitting iterations reached and no convergence achieved If necessary, increase the maximum number of iterations allowed (see Extrapolation)
Positive gradient during estimation of sigma Initial guess at sigma before fitting the sigmoid tail was poor If necessary, change the tail-fitting limits (see Extrapolation)
POLINT failed Two q values in the extrapolated dataset were identical within rounding errors Adjust the input data accordingly

 

4.4.2 Common Errors
 
Error message Cause Remedy
Back-extrapolate channel optimisation has failed Problems with back-extrapolation (e.g. due to no upturn in intensity as the beamstop is approached) Change the start of data limit (see Extrapolation)
Negative intensities in experimental data, or Data contains negative intensities The scattering data contains negative intensities Edit the values or add a constant to the data using OTOKO
X axis should be a static image dynamic found, or Expecting static Q axis received dynamic The q-axis file contains multiple frames Check that the correct file is being used (if necessary a "dummy" q-axis file can be generated with the OTOKO .XAX command
READASCII_ returning IRC=0 There is a problem with the file that the program is attempting to access (check the Message Window for any additional details)  
Exceeded maximum number of steps for F.T. The maximum number of steps for the Fourier transform is 511 Change the maximum value of d and/or the step in d in the Edit - Parameters... - Transform tab
More than 2048 channels in extrapolation The subroutines for generating a volume fraction profile cannot handle data with more than 2048 q-points Adjust the input data accordingly
